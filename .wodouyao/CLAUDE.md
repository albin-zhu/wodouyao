## You are on a Wodouyao canvas

Other agent terminals are likely active in this same workspace. They share
a task board with you. Use the `wodouyao` CLI to collaborate:

- `wodouyao peers` — list connected peers
- `wodouyao task list` — current task board for this workspace
- `wodouyao task next [--role X]` — find pickable work matching your role
- `wodouyao task claim <id>` — atomically take ownership
- `wodouyao task done <id>` — mark complete
- `wodouyao send <peer> "text" Enter` — drive a peer's input
- `wodouyao read <peer>` — read a peer's recent output

Run `wodouyao` with no args for the full subcommand list.

## Working a task

```sh
# Pull a task tagged for your role (or any task if you have no role)
task=$(wodouyao task next --role "$MY_ROLE") || exit 0  # exit 3 = nothing
tid=$(echo "$task" | python3 -c 'import json,sys;print(json.load(sys.stdin)["id"])')
wodouyao task claim "$tid" || exit 0  # exit 5 = lost the race; try again
# … do the work …
wodouyao task done "$tid"
```

## Conventions

- **Don't invent your own todo list.** The shared task board is the source
  of truth; record progress there so other agents and the human PM can see.
- **Claim before working.** `task next` only queries — multiple agents
  reading at the same instant will all see the same task. `claim` is the
  atomic gate.
- **Append to acceptance criteria as you discover them**, don't rewrite
  the subject. Use `wodouyao task update <id> --add-acceptance "..."`.
- **PRDs live as canvas notes.** A note tagged as a PRD can be parsed by
  the PM terminal into a task tree.
